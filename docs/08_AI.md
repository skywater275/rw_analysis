### MissionEngine 调试日志 (2026-06-07 从游戏运行日志验证)

MissionExecutor (n.d) 在执行 AITask 时输出以下日志:

```
MissionEngine:triggerLog:firstActivation: move at:<tick> for teamId:<X> to targetId:<Y> (#units:<Z>)
```

| 字段 | 含义 | 示例值 |
|------|------|--------|
| `at:<tick>` | 触发时的游戏tick | `6`, `261`, `7125` |
| `teamId:<X>` | AI所属队伍 (0=A..9=J) | `0`, `1`, `3` |
| `targetId:<Y>` | 目标路点ID (地图trigger定义) | `1`, `2`, `3` |
| `#units:<Z>` | 实际移动的AI单位数 | `0`=无可用单位, `>0`=实际数量 |

对比 replay_analysis 的 rc 块:
- `#units:0` 的 move 事件 → hasAU=1 的 rc 块, 无 unit_name
- `#units:>0` 的 move 事件 → hasAU=1 的 rc 块, 有坐标

这意味着 PvAI 回放中的匿名 move 事件可以追溯到具体的 AI 触发器,
为区分 AI 移动 vs 玩家移动提供了依据。

## AITaskQueue — `n.b` (1KB) ★NEW

```
字段:
  a (utility.m)   — 任务列表 (ArrayList<n.a>)
  b (boolean)     — 全部激活标志

方法:
  <init>()        — 初始化空队列
  a(n.a) → void   — 添加任务到队列
  a() → boolean   — 是否有待处理任务 (队列非空)
  b() → boolean   — ★ 所有任务是否全部激活中
                    遍历所有任务:
                      如果任一任务 j=false (未激活) → b=false, return false
                      如果 this.b 已经为 false → return false
                    所有任务都 j=true → return true
                    用途: 检测波次是否全部完成 (全部激活→已完成)
```

**用途**: 管理当前波次的所有任务。`b()` 方法用于判断波次是否应该结束：当所有任务都标记为已完成(j=true)时返回true，否则返回false。

```
   a. 每帧检查活跃任务
   b. n.b.b() → 所有任务是否全部完成?
   c. 如果全部完成 → 波次结束 → 触发 event_waveEnd
   d. 准备下一波次

6. 波次进阶:
   a. A *= (1 + r * 0.1)  — 难度递增
   b. 单位数量/质量提升
   c. 检测最终波次/无限模式
```

## 随机位置生成算法 (从 n.i.a(float,float) 反编译)

```
Java 伪代码:
  int seed = 0;
  for each SpawnEntry in b:
    for i in 0..entry.count:
      unit = entry.type.createInstance();
      
      // 位置: 基础坐标 + 随机扩散
      unit.eo = baseX + f.a(-85, 85, seed + 0);  // X: ±85px
      unit.ep = baseY + f.a(-85, 85, seed + 1);  // Y: ±85px
      
      // 朝向: 随机角度
      unit.cg = f.a(-180, 180, seed + 2);  // -180 ~ +180 度
      
      seed += 3;  // 每个单位用3维随机种子
      
      unit.bX = team;          // 设置所属队伍 (运行时类型为d)
      unit.bW = spawnPoint;    // 记录出生点
      spatialGrid.a(unit);     // 加入空间网格
      unit.S();                // 激活单位
```

`f.a(int, int, int)` 是游戏框架的 `Random.nextInt(min, max)` — 3参数版本的LCG随机数生成器，第三个参数是不同的种子偏移量，确保X/Y/旋转使用独立随机序列。
